# 장르별 플레이북

## 사용법

과제의 지배적인 목적에 맞는 구조 하나를 고른다. 아래 화살표는 사고 순서이며 고정 목차가 아니다. 글의 길이와 독자에 따라 단계는 합치거나 생략한다.

## 메커니즘·비즈니스 모델 분석

`대상의 역할 또는 구체적 장면 → 실제 가치 교환 → 행동이나 상태가 바뀌는 지점 → 반복 경로 → 이를 지탱하는 시스템 → 한계 → 압축된 판단`

- 사람이 실제로 마주하는 선택이나 작동 장면이 핵심 인과를 보여줄 때 사용한다.
- 계약 형식과 실제 행동 구조, 표면의 상품과 지불하는 가치, 한 번의 행동과 반복 구조를 구분한다.
- 사용자가 대안을 알면서도 선택한다면 무지로 설명하지 말고 맥락에 따른 효용, 마찰, 전환 비용, 습관을 검토한다.
- 심리적 설명만으로 사업 전체를 덮지 말고 공급, 품질, 단위 경제성, 유통, 규모, 규제 같은 시스템 조건으로 연결한다.
- 선택 장면이 중요하지 않은 산업에는 인간 심리나 소비 장면을 억지로 넣지 않는다.

## 기업·투자 리서치

`핵심 판단 → 미래를 좌우하는 변수 → 수익 엔진과 경제성 → 경쟁우위와 재투자 → 반대 증거와 실패 경로 → 가격·가치 구분 → 무효화 조건과 관찰 지표`

- 기업 설명을 백과사전식으로 늘리지 말고 3~5년 뒤 결과를 바꿀 소수의 변수에 집중한다.
- 사업의 질과 증권의 매력도를 분리한다. 좋은 회사가 언제나 좋은 투자는 아니다.
- 경영진의 말보다 자본 배분, 인센티브, 실제 실행 기록을 우선한다.
- 시장 규모는 회사가 실제 접근할 수 있는 경로와 점유 조건으로 좁힌다.
- 밸류에이션은 전제, 계산, 민감도, 현재 가격을 구분한다.
- 가장 강한 반대 논리와 결론이 틀렸음을 먼저 보여줄 지표를 포함한다.

## 시장·정책 인과 분석

`현상과 기간 정의 → 가능한 원인들 → 각 원인의 관측 가능한 예측 → 실제 증거 비교 → 주원인과 보조 원인 → 적용 조건과 다음 지표`

- 시점, 비교 기준, 분모를 먼저 고정한다.
- 함께 움직였다는 이유만으로 인과를 단정하지 않는다.
- 여러 설명이 양립하면 하나를 억지로 제거하지 말고 기여도와 조건을 구분한다.
- 정책의 발표, 시행, 집행, 실제 효과가 나타나는 시점을 나눈다.

## 기술 설명

`직접 답 또는 정의 → 입력과 출력 → 내부 과정 → 구체적 예 → 일반화 조건 → 병목과 실패 모드 → 단순화의 경계`

- 질문의 잘못된 전제가 있으면 먼저 바로잡고 그 위에서 설명한다.
- 명칭보다 데이터나 신호가 어떤 경로로 변환되는지 보여준다.
- 교육용 단순화와 실제 시스템의 차이를 표시한다.
- 새 입력, 배포 환경, 데이터 분포가 달라질 때 무엇을 다시 해야 하는지 설명한다.
- 비유는 실제 구성 요소와 대응할 때만 쓰고, 대응이 끊기는 지점도 밝힌다.

## 논증·비평

`주장 → 숨은 전제 → 가장 강한 근거 → 가장 강한 반론 → 증거가 가르는 지점 → 조건부 판단`

- 약한 상대를 세우지 말고 실제로 설득력 있는 반론을 다룬다.
- 가치 판단과 경험적 주장을 분리한다.
- 반론을 소개하는 데 그치지 말고 어떤 사실이 두 설명을 가르는지 말한다.
- 결론의 확신도는 증거의 강도에 맞춘다.

## 업무 메시지

`목적 또는 요청 → 행동에 필요한 맥락 → 결정 사항·기한·다음 단계`

- 첫 문단에서 받는 사람이 무엇을 알아야 하거나 해야 하는지 밝힌다.
- 예의는 유지하되 의례적 인사와 배경 설명으로 요청을 숨기지 않는다.
- 수신자, 합의, 일정, 권한, 약속을 추정하지 않는다.
- 약속의 내용이 주어지지 않았으면 요청에서 직접 따라오는 최소 약속만 쓰고, 이후 활동·성과·기한을 새로 만들지 않는다.
- 짧은 메시지에는 분석문의 반론과 결론 구조를 이식하지 않는다.

## 프롬프트·지침 문서

`목표 결과 → 입력과 범위 → 우선순위와 제약 → 필요한 절차 → 완료 조건과 검증 → 출력 형식`

- 서로 겹치는 규칙은 상위 판단 기준으로 통합한다.
- 모든 상황을 열거하지 말고 예외를 판단할 원칙을 준다.
- 결과의 모양뿐 아니라 사실 확인과 실패 시 행동을 정의한다.
- 규칙끼리 충돌하면 우선순위가 드러나게 한다.

## 에세이·문학적 산문

`구체적 장면 또는 지배적 이미지 → 의미의 확장 → 긴장이나 변화 → 절제된 귀결`

- 장면, 리듬, 여백이 의미를 운반하면 결론을 초반에 설명하지 않는다.
- 하나의 정확한 이미지를 여러 익숙한 비유보다 우선한다.
- 생산적인 모호성을 해설로 제거하지 않는다.
- 모든 문단을 주장과 근거로 환원하지 않는다.
- 실화·자전적 산문에서는 제공되지 않은 장면, 대화, 감정, 사건의 순서와 결과를 만들지 않는다.

## 기존 글 편집

`요청 범위 확인 → 강한 부분 식별 → 오류와 취향 구분 → 최소 수정 → 연결부 검증`

- 원고의 중심 판단과 고유한 리듬을 먼저 찾는다.
- 한 문장을 바꿨으면 앞뒤의 주어, 시제, 인과, 어조가 이어지는지 확인한다.
- 사용자가 더 짧게 해달라고 해도 근거나 조건을 없애 결론만 남기지 않는다.
- 번역이 중심인 작업에서는 원문 충실성을 우선하고 설치된 번역 스킬이 있으면 함께 사용한다.
