# 범용성 전향 테스트 매트릭스

## 목적

스킬을 수정할 때 서로 다른 사고 구조를 요구하는 과제에 새 대화로 적용한다. 기준 원고의 문구를 재현하는지보다 과제에 맞는 구조를 고르고 휘의 판단 밀도와 절제를 유지하는지 본다.

## 실행 원칙

1. 각 과제는 이전 출력과 의도한 답을 보여주지 않은 상태에서 실행한다.
2. 사용한 플레이북, 초안, 검수표 점수, 수정 이유를 별도로 기록한다.
3. 사실 확인이 필요한 과제는 실제 원자료를 제공하거나 접근 가능한 1차 자료를 사용한다.
4. 검수자는 내용의 정답과 문체를 분리해 평가한다.
5. 한 사례의 실패를 특정 표현 금지로 막기 전에 다른 사례에도 적용되는 개념을 찾는다.

## 과제

| 유형 | 시험 질문 | 기대 구조 | 주요 실패 신호 |
|---|---|---|---|
| 개념 설명 | `semantic representation retention이 무엇인가?` | 정의→구체적 예→작동→경계 | 회사 정의, 소비자 심리, 반복 결제 삽입 |
| 기술 분석 | `AI 기반 반도체 설계는 새로운 디자인에도 추가 작업 없이 작동하는가?` | 직접 답→입출력→일반화 조건→병목 | 인간 선택과 반복 결제 비유로 전개 |
| 시장 인과 | `정제마진과 유가가 디커플링될 수 있는 이유는 무엇인가?` | 기간·분모→후보 원인→증거→조건 | 단일 원인을 성급히 본질로 명명 |
| 기업·투자 | `노보노디스크의 경쟁우위는 5년 뒤에도 지속 가능한가?` | 핵심 변수→수익 엔진→해자→재투자→반증 | 소비 장면 하나로 기업 전체 설명 |
| 논증 | `미국 국채 금리 상승은 항상 달러 강세를 만드는가?` | 전제 교정→전달 경로→반대 경로→조건부 판단 | 장면 묘사 후 압축 명제로만 종료 |
| 메커니즘 | `연재형 유료 미리보기는 어떻게 반복 결제를 만드는가?` | 장면→선택 변화→반복 경로→시스템 조건 | 기준 원고의 고유 문구와 문단 순서 복제 |
| 업무 메시지 | `행사 참가를 수락하고 영수증과 100단어 다짐문을 보내겠다는 영문 이메일을 작성하라.` | 목적→필요 맥락→첨부·다음 단계 | 한국어식 직역, 주어지지 않은 약속, 투자 리스크·반론 추가 |
| 프롬프트 | `기업의 미래 예측 가능성을 검증하는 리서치 프롬프트를 작성하라.` | 결과→범위→우선순위→검증→출력 | 중복 규칙 나열, 스타일이 목표를 압도 |
| 편집 | `이 초안의 논리는 유지하고 과장과 중복만 줄여라.` | 강한 부분 보존→국소 수정→연결 검증 | 전면 재작성, 고유 표현 평준화 |
| 에세이 | `졸업 전 마지막 바둑 대회를 선택한 경험을 짧은 산문으로 써라.` | 제공된 장면·이미지→긴장→변화→여운 | 제공되지 않은 장면·대화·결과 창작, 보고서형 소제목 |

## 합격 기준

- 모든 결과가 `review-rubric.md`의 필수 실패 조건 없이 16점 이상이다.
- 기업 정의, 선택 순간, 번호 목록, 본질 명명, 불완전문장 결론이 과제와 무관하게 반복되지 않는다.
- 메커니즘 분석에서는 인간의 선택과 반복 구조가 보이지만 기술·메시지·에세이에서는 각 장르의 구조가 우선한다.
- 한국어와 영어 모두 직역투가 아니라 같은 수준의 직접성, 밀도, 절제를 보인다.
- 수정 작업에서 원문의 강한 문장과 형식이 보존된다.
