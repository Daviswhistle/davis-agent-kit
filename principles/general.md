# 기본 원칙

## 목적

작업을 빠르게 끝내는 것보다, 다음에 같은 실수를 반복하지 않는 구조를 만든다.

## 공통 기준

1. 추정하지 말고 확인한다.
2. 형식은 결과물의 일부다.
3. 사용자가 지적하기 전에 전면 검토한다.
4. 긴 작업은 중간 산출물과 진행표를 남긴다.
5. 좋은 결과물은 예시로 남기고, 실패한 결과물은 체크리스트로 바꾼다.
6. 공개 가능한 지식과 개인적인 판단 기록을 분리한다.
7. 한 번의 정답보다 재현 가능한 절차를 우선한다.

## 싫어하는 실패 유형

- 원문을 충분히 보지 않고 그럴듯하게 추정하는 것
- 사용자가 이미 지적한 문제를 다시 반복하는 것
- 형식 요구를 부가 요구처럼 취급하는 것
- 검수 없이 초안을 최종본처럼 내는 것
- 긴 문서를 한 번에 처리하다가 누락을 만드는 것
