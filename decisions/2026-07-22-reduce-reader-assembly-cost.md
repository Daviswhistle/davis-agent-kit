# 게시형 아티클에서 독자의 조립 비용을 검수한다

날짜: 2026-07-22

결정: `writing-quality`의 게시형 HTML 아티클 프로필에 한국어 화면 문구 압축 검수와 독자의 조립 비용 기준을 추가한다. 중요한 시각 요소는 비교 대상·기준점·방향·결과를 1차 위계에 두고, 가림 테스트 또는 콜드 리드로 핵심 명제가 바로 전달되는지 확인한다. 사용자가 수정을 요구하면 진단만 제시하지 않고 실제 변경된 산출물을 먼저 보여 준다.

배경: 한국어 원문으로 시작한 GLP-1 아티클에서도 카드 문구로 압축하는 과정에서 `감소 처방 대상`처럼 원문에 없던 번역투와 명사 나열이 생겼다. 사용량 비교 패널은 큰 숫자와 작은 보장·비보장 레이블로 정보를 흩어 놓아 독자가 비교 조건과 64.3%의 방향을 직접 조립해야 했다. 비용과 장기 편익의 귀속도 네 개의 병렬 카드로 나뉘어 인과관계가 한눈에 보이지 않았다. 기존 프로필은 시각 요소가 논리 기능을 가져야 한다고 규정했지만, 정보가 존재하는 것과 핵심 명제가 즉시 이해되는 것을 구분하지 못했다.

대안:
1. 문제가 된 문구와 레이아웃을 개별 금칙 사례로 추가한다.
2. 모든 시각 요소에 고정된 카드·막대·흐름도 템플릿을 강제한다.
3. 한국어 압축 변형, 비교축, 1차 시각 위계, 관계에 맞는 표현 형식, 가림 테스트, 결과 우선 수정이라는 일반 원칙과 전향 테스트를 추가한다.

이유: 세 번째 대안은 특정 GLP-1 사례를 외우지 않고 다른 주제의 카드, 도표, 흐름도에도 적용할 수 있다. 고정 템플릿은 관계가 다른 정보를 같은 형식에 밀어 넣는 새 실패를 만들 수 있다. 핵심 명제를 한 문장으로 고정하고 독자가 작은 문구를 조립하지 않아도 되는지 확인하면 표현 형식은 내용에 맞게 달라질 수 있다.

변경:
- `skills/writing-quality/SKILL.md`: 한국어 화면 문구를 별도 편집 산출물로 검수하고, 독자의 조립 비용과 실제 결과 우선 원칙을 연결한다.
- `skills/writing-quality/references/publishable-html-article.md`: 한국어 원문 압축, 비교 범주와 행위자, 하나의 시각 요소에 하나의 명제, 관계에 맞는 형식, 가림 테스트, 콜드 리드를 추가한다.
- `skills/writing-quality/references/review-rubric.md`: 작은 보조 문구 없이 이해되는지와 실제 수정 결과를 필수 검수 항목으로 둔다.
- `skills/writing-quality/references/test-matrix.md`: 한국어 화면 문구 압축, 비교축·행위자, 시각적 위계, 관계 시각화, 수정 결과 제시 과제를 추가한다.
- `skills/writing-quality/references/reader-first-information-design-examples.md`: 실패를 특정 문구가 아니라 개념으로 일반화한 전후 사례를 둔다.
- 정적 계약 테스트는 새 프로필·예시·검수·전향 테스트의 연결을 확인한다.

검증: Python 단위 테스트와 레포 전체 검증을 실행한다. 정적 테스트는 문서 계약과 번들 연결만 보장하므로, 실제 품질 검증에서는 새 대화로 전향 과제를 수행하고 시각 과제에 가림 테스트 또는 콜드 리드를 적용한다.

나중에 다시 볼 조건: 독립 검수자 없이도 자동으로 시각적 명제를 안정적으로 추출할 수 있는 평가 도구가 도입되거나, 게시형 아티클 외의 슬라이드·대시보드·앱 UI에서도 같은 실패가 반복돼 범용 정보 디자인 스킬로 분리할 필요가 생길 때 재검토한다.
