# 모델 중립적 위임 원칙

날짜: 2026-07-16

결정: 공개 전역 `AGENTS.md`에는 주 세션의 최종 책임, 선택적 위임, 완료 근거, 증거 기반 검증에 관한 모델 중립적 원칙만 둔다. 특정 모델, reasoning effort, 공급자, 비용 등급은 전역 기본값으로 강제하지 않는다. Max·Medium 같은 구체적 자원 배치는 사용자별 런타임 설정에 맡기고, 별도 위임 스킬은 반복 실패와 평가 근거가 생기기 전에는 추가하지 않는다.

배경: 높은 추론 수준의 주 세션이 실행량이 큰 작업을 중간 추론 수준의 작업자에게 위임하고 별도 문맥에서 검토하게 하는 구조는 품질과 맥락 효율을 높일 수 있다. 그러나 공개 키트의 사용자는 요금제, 모델 접근권, 지연 허용도, 공급자, 로컬 실행 환경이 서로 다르다. 특정 고비용 모델 구성을 전역 기본값으로 두면 범용성을 해치고, 하위 에이전트가 없는 환경에서는 지침이 실행 불가능해진다.

대안:

1. 최고 추론 수준의 모델을 모든 사용자의 주 에이전트로 강제하고 중간 추론 수준의 작업자를 기본 호출한다.
2. 위임 판단과 검증 절차를 즉시 별도 스킬로 만든다.
3. 전역 원칙을 추가하지 않고 각 사용자가 런타임 설정만 관리한다.

이유: 주 세션이 의도 해석과 최종 책임을 유지하고, 위임이 실질적 이점이 있을 때만 목표·범위·제약·완료 근거를 명시해 맡기며, 작업자의 완료 선언을 실제 증거로 검증한다는 원칙은 모델과 도구가 바뀌어도 유지된다. 반면 구체적인 모델과 reasoning effort는 비용과 기능 변화에 민감한 구현 선택이다. 현재 Codex의 하위 에이전트 기능으로 먼저 평가할 수 있으므로, 검증되지 않은 절차를 스킬로 고정하지 않는 편이 키트의 단순성과 변경 측정 원칙에도 맞는다.

나중에 다시 볼 조건:

- 위임 계약 누락, 잘못된 병렬화, 완료 선언의 무비판적 수용이 여러 작업에서 반복될 때
- 특정 위임 절차가 모델과 도구를 넘어 안정적으로 재현된다는 평가가 쌓일 때
- 커스텀 작업자 설치와 역할별 설정을 공개 기능으로 제공할 충분한 수요와 검증 계약이 생길 때
- Codex의 네이티브 하위 에이전트 설정이나 reasoning effort 동작이 바뀔 때
