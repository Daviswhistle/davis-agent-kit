# 범용성 전향 테스트 매트릭스

## 목적

스킬을 수정할 때 서로 다른 사고 구조를 요구하는 과제에 새 대화로 적용한다. 기준 원고의 문구를 재현하는지보다 과제에 맞는 구조를 고르고 이 스킬의 판단 밀도와 절제를 유지하는지 본다.

## 실행 원칙

1. 각 과제는 이전 출력과 의도한 답을 보여주지 않은 상태에서 실행한다.
2. 사용한 플레이북, 초안, 검수표 점수, 수정 이유를 별도로 기록한다.
3. 사실 확인이 필요한 과제는 실제 원자료를 제공하거나 접근 가능한 1차 자료를 사용한다.
4. 검수자는 내용의 정답과 문체를 분리해 평가한다.
5. 한 사례의 실패를 특정 표현 금지로 막기 전에 다른 사례에도 적용되는 개념을 찾는다.
6. 관계 설계와 문장 구현을 따로 평가한다. 전략이 맞아도 문장이 부자연스러우면 실패로 기록한다.
7. 호의 기반 협력 요청과 의무·제재·평가 메시지를 서로 다른 과제로 시험해, 새 원칙이 모든 업무 메시지에 과적용되지 않는지 본다.

## 과제

| 유형 | 시험 질문 | 기대 구조 | 주요 실패 신호 |
|---|---|---|---|
| 개념 설명 | `semantic representation retention이 무엇인가?` | 정의→구체적 예→작동→경계 | 회사 정의, 소비자 심리, 반복 결제 삽입 |
| 기술 분석 | `AI 기반 반도체 설계는 새로운 디자인에도 추가 작업 없이 작동하는가?` | 직접 답→입출력→일반화 조건→병목 | 인간 선택과 반복 결제 비유로 전개 |
| 시장 인과 | `정제마진과 유가가 디커플링될 수 있는 이유는 무엇인가?` | 기간·분모→후보 원인→증거→조건 | 단일 원인을 성급히 본질로 명명 |
| 기업·투자 | `노보노디스크의 경쟁우위는 5년 뒤에도 지속 가능한가?` | 핵심 변수→수익 엔진→해자→재투자→반증 | 소비 장면 하나로 기업 전체 설명 |
| 논증 | `미국 국채 금리 상승은 항상 달러 강세를 만드는가?` | 전제 교정→전달 경로→반대 경로→조건부 판단 | 장면 묘사 후 압축 명제로만 종료 |
| 메커니즘 | `연재형 유료 미리보기는 어떻게 반복 결제를 만드는가?` | 장면→선택 변화→반복 경로→시스템 조건 | 기준 원고의 고유 문구와 문단 순서 복제 |
| 업무 메시지 | `행사 참가를 수락하고 영수증과 100단어 다짐문을 보내겠다는 영문 이메일을 작성하라.` | 직접 실행형: 목적→필요 맥락→첨부·다음 단계 | 한국어식 직역, 주어지지 않은 약속, 관계형 서론으로 요청을 숨김 |
| 업무 메시지·관계형 | `여러 부서가 디자인 수정 의견을 따로 보내 일정이 늦어진다. 앞으로는 1차 의견을 한 문서로 취합해 달라는 메시지를 작성하라.` | 관계 가치→완성도 높은 최종안을 주고 싶은 선의→분산 의견이 최종본 전달을 늦추는 문제→취합 요청→구체적 이익→지속되는 지원 | 내부 재작업만 호소, `귀사에서 확인하는 시점` 같은 억지 수신자 관점, `아쉽습니다` 반복, 조건부 호의 |
| 업무 메시지·원형 변환 | `오후 늦게 도착하는 대량 화물을 오전에 반입해 달라는 불평형 편지를 호의 기반 수신자 관점 설득으로 고쳐라.` | 감사·관계→좋은 서비스 의지→혼잡이 수신자 결과를 막는 구조→오전 반입 제안→대기·당일 처리 이익→협조와 무관한 호의 | 발신자 야근을 중심에 둠, 공손한 명령만 추가, 수신자 이익을 마지막에 덧붙임 |
| 업무 메시지·경계 | `연체 대금과 서비스 중단 가능성을 통지하라.` | 존중을 갖춘 직접성: 미납 사실→계약상 결과→기한·가능한 해결 경로 | 채권 회수를 `귀사를 위한 조언`으로 위장, 추측성 공감, 조건부 거래를 무조건적 호의로 표현 |
| 업무 메시지·성과 | `목표를 달성하지 못한 직원에게 개선 피드백을 작성하라.` | 공정한 성과 피드백: 사실→책임→확인된 강점→개선 행동·대화 | 실적 책임 면제, 칭찬 샌드위치, 관리 지시를 관계형 설득으로 위장 |
| 문장 교정 | `“귀사에서 최종본을 확인하는 시점이 늦어질 수 있어 아쉽습니다”를 뜻은 유지하고 자연스럽게 고쳐라.` | 실제 사건을 직접 표현: `최종본 전달도 늦어질 수 있습니다` | 추상 명사 유지, 감정어만 교체, 관계 전략을 이유로 어색함 방치 |
| 분석 글 제목 | `GPU 조달 뒤 전력 접속이 새 제약으로 부상한다는 글의 제목을 고르라.` | 기존 병목을 부정하지 않고 이동 방향과 시간성을 표현 | `진짜 병목은 GPU가 아니다`처럼 근거보다 강한 양자택일 |
| 한국어 자료 재서술 | `“유틸리티 전망을 거칠게 합산했고 정보의 비대칭을 보여준다”를 뜻은 유지하고 자연스럽게 고쳐라.` | 실제 행위와 불확실성을 직접 표현 | 원문의 부사·추상 명사구를 한국어에 그대로 옮김 |
| 수치 카드 | `<3%`, `>5년`, `1,000%+`를 한국어 대중 원고의 카드로 표시하라. | `3% 미만`, `5년 초과`, `1,000% 이상`; 숫자와 비교어를 한 의미 단위로 표시 | 기호식 표기 유지, `이상`을 별도 배지나 버튼처럼 분리 |
| 게시형 HTML | `6단계 흐름도와 큰 수치 카드를 CMS의 좁은 본문 안에서도 읽히게 설계하라.` | 실제 컨테이너 폭별 재배치→긴 한국어 항목 스트레스 테스트→기계 검사와 스크린샷 검수 | 뷰포트 미디어 쿼리만 사용, 한 줄 고정, 글자 넘침, 자동 검사만 통과 |
| 패턴 수정 회귀 | `한 문장에서 불필요한 “뜻하지 않습니다” 구조가 지적된 완성 원고와 HTML·TXT·미리보기를 수정하라.` | 실패 유형 일반화→전체 산출물 검색→최종 HTML 재렌더링→일치 확인 | 지적된 한 문장만 고치고 전체 해결을 선언 |
| 프롬프트 | `기업의 미래 예측 가능성을 검증하는 리서치 프롬프트를 작성하라.` | 결과→범위→우선순위→검증→출력 | 중복 규칙 나열, 스타일이 목표를 압도 |
| 편집 | `이 초안의 논리는 유지하고 과장과 중복만 줄여라.` | 강한 부분 보존→국소 수정→연결 검증 | 전면 재작성, 고유 표현 평준화 |
| 에세이 | `졸업 전 마지막 바둑 대회를 선택한 경험을 짧은 산문으로 써라.` | 제공된 장면·이미지→긴장→변화→여운 | 제공되지 않은 장면·대화·결과 창작, 보고서형 소제목 |

## 합격 기준

- 모든 결과가 `review-rubric.md`의 필수 실패 조건 없이 16점 이상이다.
- 기업 정의, 선택 순간, 번호 목록, 본질 명명, 불완전문장 결론이 과제와 무관하게 반복되지 않는다.
- 메커니즘 분석에서는 인간의 선택과 반복 구조가 보이지만 기술·메시지·에세이에서는 각 장르의 구조가 우선한다.
- 한국어와 영어 모두 직역투가 아니라 같은 수준의 직접성, 밀도, 절제를 보인다.
- 수정 작업에서 원문의 강한 문장과 형식이 보존된다.
- 관계형 요청에서는 발신자의 고충이 아니라 수신자가 원하는 결과와 발신자의 선의가 글의 중심이 된다.
- 직접 통지·성과 평가·안전 명령에서는 관계형 설득을 억지로 사용하지 않는다.
- 수신자 관점을 넣더라도 추상적인 명사구와 감정어가 늘어나지 않는다.
- 게시형 HTML 과제는 `publishable-html-article.md`의 추가 검수를 통과하고, 실제 컨테이너 폭과 최종 산출물 일치에 대한 증거를 남긴다.
