# 독자 우선 정보 설계 전후 사례

## 목적

게시형 아티클에서 문장과 시각 요소가 독자에게 불필요한 해석·탐색·조립 작업을 넘기지 않도록 하는 기준 사례다. 특정 문구를 금칙어로 외우지 않는다. 각 사례에서 실패를 만든 개념을 찾아 다른 주제와 형식에도 적용한다.

## 1. 한국어 원문도 압축 과정에서 망가질 수 있다

### 실패

```text
감소
처방 대상
```

원문이 한국어였어도 카드 크기와 대칭을 맞추는 과정에서 조사와 동사가 사라지고 명사 관계가 뒤집혔다. `감소`가 무엇을 꾸미는지, 누가 무엇을 줄였는지 알 수 없다.

### 개선

```text
보장 기준 강화
→ 대상 환자 수 감소
```

짧게 쓰더라도 원인과 결과를 보존한다. 카드 레이블은 원문에서 자동 추출하는 요약이 아니라 별도의 한국어 문장으로 검수한다.

## 2. 같은 종류의 대상끼리 비교한다

### 실패

> 공공보험에서는 장기 편익보다 예산창이 먼저 보인다.

`장기 편익`은 정책 효과이고 `예산창`은 평가 기간이다. 비교 범주가 다르고, 누가 판단하는지도 없다.

### 개선

> 정부의 보장 결정에서는 장기적인 건강 개선을 인정하더라도, 예산 추계 기간 안에 발생하는 약제비 증가가 더 크게 반영될 수 있다.

효과와 기간을 직접 비교하지 않는다. 실제 행위자와 평가 구조를 밝히고, 필요하면 같은 기간의 약제비 증가와 의료비 절감을 비교한다.

## 3. 결론이 숫자보다 먼저다

### 실패 구조

```text
[1,260.0일분]   [64.3% 격차]   [766.8일분]
  작은 글씨: 보장                작은 글씨: 비보장
```

가장 중요한 비교 조건이 가장 작다. 독자는 숫자를 본 뒤 작은 레이블을 찾고, 기준 방향을 추론하고, 격차의 뜻을 다시 조립해야 한다.

### 개선 구조

```text
보장 플랜의 GLP-1 공급일수는 비보장 플랜보다 64.3% 많았습니다

비보장 플랜  ███████████          767일
보장 플랜    ██████████████████  1,260일
```

결론형 제목이 핵심 명제를 먼저 말하고, 막대와 숫자가 그 문장을 증명한다. 작은 범위 문구와 각주를 가려도 비교 대상과 방향을 알 수 있어야 한다.

## 4. 관계를 독자가 조립하게 하지 않는다

### 실패 구조

`지금 부담하는 것`, `나중에 얻는 것`, `가입자를 오래 보유하면`, `보험 변경이 잦으면`을 네 개의 병렬 카드에 둔다. 정보는 모두 있지만 어떤 조건이 어떤 결과를 만드는지 독자가 연결해야 한다.

### 개선 구조

```text
가입자가 같은 보험에 남음
보험자 A: 지금 약제비 부담 → 보험자 A: 나중의 의료비 절감
보장할 유인 커짐

가입자가 다른 보험으로 이동
보험자 A: 지금 약제비 부담 → 보험자 B: 나중의 의료비 절감
보장할 유인 약해짐
```

핵심 질문을 `비용을 낸 주체와 편익을 얻는 주체가 같은가`로 좁히고 두 경로만 비교한다. 시간과 주체의 이동은 병렬 카드가 아니라 흐름도로 표현한다.

## 5. 수정 요청에는 결과부터 보여 준다

### 실패

사용자가 시각 요소를 이해하기 어렵다고 지적했는데, 문제 진단과 디자인 원칙만 길게 설명한다.

### 개선

1. 실제 HTML이나 원본 파일을 수정한다.
2. 수정된 미리보기나 산출물을 먼저 보여 준다.
3. 무엇이 달라졌고 어떤 검증을 했는지 짧게 설명한다.

작업의 목적이 결과물 수정이라면 설명은 산출물을 대신할 수 없다.
