# 블라인드 검수표

## 채점

각 항목을 0~2점으로 평가한다.

- 0점: 빠졌거나 목적을 해친다.
- 1점: 존재하지만 약하거나 일부만 작동한다.
- 2점: 글의 목적에 맞게 정확히 작동한다.

1. 질문이나 요청에 직접 답한다.
2. 구조가 과제의 장르, 독자, 사용 환경에 맞다.
3. 사실, 추론, 가정, 미확인 영역의 경계가 보인다.
4. 결론까지 이어지는 핵심 인과 단계가 빠지지 않았다.
5. 필요한 구체적 사실, 숫자, 비교, 장면, 과정이 있다.
6. 중요한 반대 근거, 경계 조건, 불확실성이 중요도에 비례해 다뤄졌다.
7. 각 문단이 하나의 역할을 맡고 논리를 앞으로 보낸다.
8. 문장이 직접적이고 정확하며 길이와 리듬이 기능에 맞다.
9. 기존 원고를 편집했다면 고유한 목소리와 강한 부분을 보존했다.
10. 결말이 반복 요약 없이 판단, 행동, 지표, 여운 중 적절한 방식으로 닫힌다.

16점 이상을 기본 통과로 삼는다. 다만 사실 오류, 출처 조작, 제공되지 않은 경험·대화·약속의 창작, 요청 형식 위반, 핵심 질문 미응답이 하나라도 있으면 총점과 무관하게 통과시키지 않는다. 투자·법률·의료처럼 판단 비용이 큰 글은 18점 이상을 목표로 한다.

## 관계 민감한 협조 요청 추가 검수

`호의 기반 협력 요청`을 선택한 경우에만 아래 여섯 항목을 0~2점으로 추가 평가한다.

1. 수신자가 실제로 원하는 좋은 결과가 구체적으로 드러난다.
2. 발신자가 그 결과를 바란다는 사실이 자기 선언이 아니라 감사, 관점, 제안, 마무리의 일관성으로 입증된다.
3. 현재 문제는 발신자의 고충이나 수신자에 대한 기소장보다, 수신자의 좋은 결과를 방해하는 장애물로 설명된다.
4. 요청한 행동이 그 결과를 실제로 개선하며, 조건부 보상·완곡한 위협·꾸며 낸 이익이 아니다.
5. 수신자가 협조하지 않더라도 기본적인 존중과 호의가 철회되지 않는다. 단, 제공할 수 없는 서비스나 예외를 약속하지 않는다.
6. 관계 전략과 별개로 문장이 자연스럽고 구체적이다. 수신자 이익을 억지로 넣은 명사화, 주어 전환, 감정어 반복이 없다.

9점 이상을 통과 기준으로 삼되, 1~4번 중 하나라도 0점이면 통과시키지 않는다. 다음은 총점과 무관한 실패다.

- 존재하지 않는 관계, 과거 기여, 칭찬, 수신자 이익을 만들어 낸다.
- 의무·제재·평가·안전 명령을 수신자를 위한 선택 가능한 조언처럼 위장한다.
- `귀사를 위해 드리는 말씀입니다` 같은 선언만 있고 글의 실제 구조는 발신자 중심이다.
- `해 주시면 잘해 드리겠습니다` 같은 조건부 거래를 무조건적인 호의로 오인한다.
- 전략이 맞다는 이유로 `귀사에서 최종본을 확인하는 시점` 같은 부자연스러운 문장을 남긴다.

관계형을 선택하지 않은 메시지도 상대의 인격이나 동기를 함부로 판단해서는 안 된다. 다만 연체 통지, 성과 평가, 안전 지시처럼 직접성이 필요한 메시지에 이 추가 점수표를 강제로 적용하지 않는다.

## 게시형 HTML 아티클 추가 검수

원고와 HTML/CSS, CMS용 코드, 미리보기 이미지가 함께 산출물인 경우 아래 항목을 0~2점으로 추가 평가한다. 1~11번은 항상 평가하고, 12번은 사용자가 기존 산출물의 수정을 요구한 작업에만 적용한다.

1. 제목이 본문이 입증한 변화·메커니즘·판단과 주장 강도를 정확히 반영한다.
2. 본문 한국어가 원자료의 문장 구조를 옮기지 않고 독자의 이해 순서로 다시 쓰였으며, 존재하지 않는 반론을 먼저 처리하게 하지 않는다.
3. 영어 자료뿐 아니라 **한국어 원문을 화면 문구로 압축**한 부분도 자연스럽다. 명사 나열, 주어 소실, 억지 대칭이 없고 비교축과 행위자가 정확하다.
4. 대중 대상 수치는 `이상`, `이하`, `초과`, `미만`으로 자연스럽게 쓰였고 숫자와 비교어가 하나의 의미 단위로 보인다.
5. 수치의 집계 대상, 관측·추정 여부, 제안·결정·시행 단계, 출처 성격이 요약과 시각 요소에도 정확히 반영됐다.
6. 각 시각 요소가 하나의 핵심 명제를 맡고, 크기·시간·이동·산술·관점 등 관계에 맞는 형식을 사용한다.
7. 핵심 명제에 필요한 비교 대상, 기준점, 방향, 결과가 1차 시각 위계에 있다. **작은 보조 문구**를 찾아 읽거나 흩어진 단서를 조립해야만 의미가 완성되지 않는다.
8. 배지·버튼·탭처럼 보이는 모양은 실제 상호작용이나 상태 의미가 있을 때만 사용하고, 평범한 설명을 가짜 인터페이스처럼 보이게 만들지 않는다.
9. 실제 아티클 컨테이너의 데스크톱·중간·모바일 폭에서 잘림, 겹침, 가로 넘침, 과도한 축소가 없고 읽는 순서가 유지된다.
10. 기계 검사와 별도로 최종 스크린샷을 사람이 읽었고, 중요한 패널은 **가림 테스트나 콜드 리드**에서 의도한 한 문장으로 설명됐다.
11. 최종 HTML, CMS용 코드, 미리보기의 내용이 일치하고, 사용자 피드백에서 일반화한 실패 유형을 전체 산출물에서 다시 검색했다.
12. 사용자가 수정을 요구한 작업이라면 진단과 계획만 제시하지 않고 **실제 수정 결과**를 먼저 보여 주었다.

최초 작성 작업은 1~11번에서 19점 이상, 수정 작업은 1~12번에서 21점 이상을 통과 기준으로 삼는다. 최초 작성처럼 수정 요청이 없으면 12번은 `해당 없음`으로 기록하고 점수와 분모에서 제외한다. 5~11번 중 하나라도 0점이면 총점과 무관하게 게시 가능 상태로 판정하지 않는다. 수정 작업에서는 12번도 0점이면 통과시키지 않는다. 검사를 실행하지 않았다면 1점으로 추정하지 말고 미검증으로 기록한다.

## 과최적화 감점

다음 패턴이 과제의 필요가 아니라 습관으로 나타나면 항목마다 1점을 감점한다.

- 모든 글이 회사나 대상의 사전적 정의로 시작한다.
- 모든 글에 소비자의 선택 순간과 반복 행동이 등장한다.
- 모든 글이 같은 수의 번호 목록으로 전개된다.
- 결론 직전에 언제나 `X는 Y를 파는 회사/장치다`라고 명명한다.
- 마지막 문단이 언제나 여러 명사구와 불완전 문장으로 구성된다.
- 기준 원고의 비유나 전환 문구가 주제와 무관하게 재사용된다.
- 짧은 메시지에도 반론, 실패 경로, 관찰 지표를 붙인다.
- 기술 설명이나 에세이에 사업 모델 용어를 억지로 넣는다.
- 정보 탐색이 필요한 긴 글에서도 소제목과 표를 무조건 피한다.

## 수정 순서

1. 사실 오류와 요청 위반
2. 결론을 바꾸는 논리와 빠진 인과
3. 범위, 주어, 용어의 불명확성
4. 중복과 구조
5. 리듬과 어휘

낮은 항목을 올리기 위해 이미 정확하고 고유한 문장을 평범하게 만들지 않는다. 감점 패턴도 문구 검색만으로 판정하지 말고 그 구조가 과제에 실제로 필요한지 판단한다.
