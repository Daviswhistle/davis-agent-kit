# 범용 글쓰기 스킬 통합

날짜: 2026-07-11

결정: `hwi-analytical-writing`과 `write-in-daehwi-style`의 유효한 규칙을 `skills/hwi-writing/` 하나로 통합한다. 공통 판단 원칙은 `SKILL.md`에 두고, 과제별 구조와 검수·전향 테스트는 참조 문서로 분리한다.

배경: 분석형 스킬은 구체적 작동 장면, 인간의 선택, 반복 구조를 설명하는 데 강했지만 하나의 비즈니스 모델 원고에서 추출한 문단 순서와 문장 지문을 대부분의 글에 요구했다. 범용 스타일 스킬은 장르 적응과 사실·추론 구분에 강했지만 원샷 작성 절차와 장르별 선택 기준이 짧았다. 두 스킬이 함께 활성화되면 어느 구조가 우선하는지도 불분명했다.

대안:

1. 두 스킬을 그대로 유지하고 분석 작업에만 전자를 추가로 적용한다.
2. 분석형 스킬을 기준으로 다른 장르 규칙을 계속 덧붙인다.
3. 공통 원칙과 장르별 가변 구조를 분리한 하나의 범용 스킬로 다시 설계한다.

이유: 세 번째 대안이 필요한 원칙을 한 곳에 통합하면서도 특정 사례의 문단 순서를 일반 규칙으로 만들지 않는다. 메커니즘 분석의 강점은 하나의 플레이북으로 보존하고, 투자·시장 인과·기술·논증·업무 메시지·프롬프트·에세이·편집은 각 목적에 맞는 구조를 선택한다.

기존 분석형 검사기의 질문형 제목, 고정 번호 목록, 특정 비유, 불완전문장 결론 점수는 이식하지 않는다. 이 지표들은 기준 원고와 닮은 정도는 측정하지만 새로운 장르에서 좋은 글인지 판정하지 못한다. 대신 사실·인과·장르 적합성·목소리 보존을 평가하는 블라인드 검수표와 서로 다른 10개 전향 테스트 과제를 사용한다.

첫 전향 테스트에서는 기술 설명이 장르에 맞게 분기됐지만, 영문 업무 메시지가 주어지지 않은 사후 활동을 약속하고 자전적 산문이 제공되지 않은 장면과 결과를 사실처럼 만드는 문제가 드러났다. 글의 완결성과 생생함을 위해 경험·대화·약속을 창작하지 않도록 공통 사실성 규칙과 두 장르의 플레이북을 보강했다.

같은 요청을 새 작업 맥락에서 다시 실행한 결과, 영문 메시지는 내용이 없는 다짐문을 자리표시자로 남겼고 자전적 산문은 제공된 선택과 그 의미 안에서 전개됐다. 사실성 규칙을 강화한 뒤에도 메시지의 간결함과 산문의 리듬은 유지됐다.

나중에 다시 볼 조건: 실제 사용에서 특정 장르가 반복해서 약해지거나, 플레이북끼리 충돌하거나, 전향 테스트를 통과하면서도 휘의 문체가 희석되는 사례가 누적될 때 구조와 검수 기준을 다시 본다.
