# 전략

이 레포의 전략은 [PHILOSOPHY.md](PHILOSOPHY.md)의 철학을 작업에 적용하기 위한 변하지 않을 소수의 판단 기준입니다.

실행물은 전략을 현재 환경에 적용하는 문서와 도구입니다. `AGENTS.md`, `GEMINI.md`, `principles/`, `checklists/`, `templates/`, `skills/`, 테스트, 폴백 규칙은 실제 작업과 검증 결과에 따라 계속 수정합니다.

## 전략 승격 기준

어떤 기준을 전략으로 올리기 전에 아래 조건을 확인합니다.

1. 여러 작업, 도구, 파일 형식에 적용된다.
2. 새 사례를 만날 때 별도 규칙 추가를 줄인다.
3. 변하지 않을 기준으로 둘 만큼 안정적이다.
4. 실행물의 방향을 정한다.
5. 작업자의 판단을 바꾼다.

## 현재 전략

1. 구체 사례에서 숨은 개념을 찾고, 같은 개념이 적용되는 다른 표면 사례까지 본다.
2. 확인 가능한 사실과 실행 결과로 판단한다.
3. 산출물은 실제 독자나 사용자가 바로 쓸 수 있는 상태로 끝낸다.
4. 기준 원본, 실행 진입점, 검증 경로가 같은 모델을 따르는지 확인한다.
5. 폴백, 테스트, 체크리스트, 스킬은 전략을 실행하는 현재 버전으로 관리한다.
