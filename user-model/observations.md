# 사용자 관찰 기록

## 2026-06-22

### 번역 작업 품질 기준

관찰:
사용자는 번역 결과물이 단순히 의미만 맞는 수준이 아니라, 바로 블로그나 문서에 붙여넣어 사용할 수 있는 상태이기를 기대한다.

근거:
룰루레몬 실적발표 transcript 번역 작업에서 화자 표기, 문단 간격, 이태릭체, 링크 유지, 주석 위치, 어색한 직역 표현, 숫자 가이던스 문장까지 반복적으로 검토하고 수정했다.

작업에 미치는 영향:
번역 요청은 초안 생성이 아니라 전체 문서 검토, 형식 검수, 기계 검색, 숫자 대조, 최종 산출물 확인까지 포함된 작업으로 본다.

반영 위치:
- `principles/reproducible-quality.md`
- `checklists/translation-qa.md`
- `skills/translation-quality/README.md`
- `skills/translation-quality/SKILL.md`

확신도:
높음

### 운영 기준 레포 갱신 방식

관찰:
사용자는 반복해서 확인된 작업 방식과 품질 기준을 운영 기준 레포에 빠르게 반영하기를 원한다.

근거:
사용자가 운영 기준을 레포와 스킬로 외부화하고, 여러 기기에서도 같은 품질 기준이 적용되기를 원한다고 반복해서 말했다.

작업에 미치는 영향:
사용자 선호, 작업 철학, 체크리스트, 스킬 운영 규칙이 반복 확인되면 `davis-agent-kit`에 반영할지 검토한다. commit, push, 원격 레포 rename처럼 상태를 바꾸는 작업은 현재 적용 중인 전역 지침과 사용자의 명시 요청을 따른다.

반영 위치:
- `AGENTS.md`
- `README.md`

확신도:
높음

### GitHub 인증 입력 방식

관찰:
사용자는 GitHub 토큰 입력이 필요한 경우, 내부 터미널 프롬프트가 아니라 실제 화면에 보이는 입력창이 뜨기를 기대한다.

근거:
GitHub push 중 내부 PTY에 `Password for ...` 프롬프트가 떴지만 사용자 화면에는 아무 창이 보이지 않았다. 이후 `GIT_ASKPASS`와 macOS `osascript` 입력창을 사용했을 때 사용자가 토큰을 입력할 수 있었고 push가 성공했다.

작업에 미치는 영향:
GitHub 인증이 필요한 작업에서는 내부 PTY 프롬프트에 의존하지 않는다. 비대화형 인증이 없으면 macOS GUI 입력창을 띄우거나, 사용자가 직접 터미널에서 실행할 명령을 안내한다.

반영 위치:
- `checklists/github-auth.md`

확신도:
높음

### 작업 방식의 절차화 선호

관찰:
사용자는 반복해서 발생한 품질 문제를 말로만 기억하는 것보다 스킬, 체크리스트, 프로젝트 규칙으로 고정하기를 원한다.

근거:
번역 스킬을 만든 뒤 실제 `codex exec` 호출로 검증하고, 실패 지점을 스킬에 반영하도록 요청했다. 이후 이를 개인 작업 기준 체계로 확장하자고 했다.

작업에 미치는 영향:
반복 실수나 중요한 선호가 드러나면 `inbox/`에 임시 기록한 뒤, 필요하면 `principles/`, `checklists/`, `skills/`로 승격한다.

반영 위치:
- `README.md`
- `AGENTS.md`
- `principles/reproducible-quality.md`
- `user-model/README.md`

확신도:
높음
