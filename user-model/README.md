# User Model

사용자와 함께 작업하면서 파악한 작업 방식, 품질 기준, 선호, 반복 피드백을 기록합니다.

목적은 에이전트가 사용자의 판단 기준과 작업 습관을 따라 사고하고 행동하는 데 필요한 정보를 구조화하는 것입니다. 사용자에 대한 사적 분석은 배제하고, 다음 작업의 품질에 영향을 주는 정보만 기록합니다.

## 기록 기준

기록해도 되는 것:

- 반복해서 드러난 작업 선호
- 결과물 품질에 직접 영향을 주는 기준
- 싫어하는 실패 유형
- 자주 쓰는 출력 형식
- 검증 방식에 대한 기대
- 공개 가능한 작업 흐름
- 투자 판단 방식
- 개인 선호와 작업 철학

기록하지 않는 것:

- 계정, 토큰, 키, 비밀번호
- 사적인 신상 정보 중 작업 재현성과 무관한 내용
- 근거 없는 성격 분석
- 사용자가 원하지 않는 사적 기록
- 비공개 원문 전문
- 제3자 민감정보
- 계약상 비공개 자료

## 기록 방식

각 관찰은 가능한 한 다음 형식으로 남깁니다.

```text
날짜:
관찰:
근거:
작업에 미치는 영향:
반영 위치:
확신도:
```

## 승격 규칙

관찰은 처음에는 `observations.md`에 남깁니다. 반복 확인되거나 작업 규칙으로 굳힐 만하면 다음 위치로 승격합니다.

- 전역 원칙: `principles/`
- 완료 전 검수: `checklists/`
- 반복 작업 절차: `skills/`
- 출력 형식: `templates/`
- 좋은/나쁜 사례: `examples/`
