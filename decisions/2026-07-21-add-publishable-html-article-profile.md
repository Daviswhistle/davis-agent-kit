# 게시 가능한 HTML 아티클 프로필 추가

날짜: 2026-07-21

결정: 새 스킬을 만들지 않고 `skills/writing-quality/references/publishable-html-article.md`를 추가한다. `writing-quality`가 주장, 한국어, 정보 구조와 시각적 의미를 소유하고, HTML/CSS 구현과 렌더링 검증이 실질적인 작업이면 기존 `software-engineering`을 함께 적용한다.

배경: 분석 원고를 단일 파일 HTML/CSS와 CMS용 코드, 미리보기 이미지로 만드는 작업에서 같은 실패가 반복됐다. 불필요한 부정형 대조, 영어식 직역, 수치 비교 기호, 제목의 과도한 양자택일, 숫자와 비교어의 시각적 분리, 실제 CMS 폭에서의 오버플로, 국소 수정 뒤 전체 해결을 선언하는 문제가 있었다. 기존 글쓰기 원칙은 자연스러운 한국어와 실제 혼동에 필요한 대조만 쓰도록 규정하지만, 게시형 아티클의 수치·시각 의미·컨테이너 렌더링·파생 산출물 일치까지 실행 계약으로 연결하지 않았다.

대안:
1. `AGENTS.md`에 모든 세부 규칙을 추가한다.
2. `publishable-article`이라는 새 스킬을 만든다.
3. 기존 `writing-quality` 안에 게시형 HTML 아티클 프로필과 추가 검수·전향 테스트를 둔다.

이유: 세 번째 대안은 전역 지침을 비대하게 만들지 않고, 이미 글쓰기 스킬이 소유한 주장·문장·장르 판단을 재사용한다. 새 스킬을 만들면 `writing-quality`와 호출 경계가 겹치고 원고 품질과 웹 검증의 책임이 분산된다. 프로필로 분리하면 게시형 산출물에만 수치 표현, 시각 문법, 실제 컨테이너 렌더링, HTML·TXT·미리보기 일치 검수를 추가할 수 있다.

검증: 프로필이 `SKILL.md`에서 참조되는지, 제목·한국어·수치·원자료 범위·시각 의미·렌더링·전수 회귀 계약을 포함하는지 정적 계약 테스트로 확인한다. 별도로 `test-matrix.md`의 제목, 한국어 재서술, 수치 카드, 좁은 CMS 컨테이너, 패턴 수정 회귀 과제를 새 대화에서 실행한다. 정적 테스트 통과는 실제 글쓰기와 렌더링 품질을 보장하지 않으므로 대표 전향 테스트와 스크린샷 검수를 완료 기준에 포함한다.

나중에 다시 볼 조건: HTML/CSS 아티클 작업이 다른 시각 문서·슬라이드·대시보드까지 확장돼 별도 라우팅과 도구 체인이 필요해지거나, `writing-quality`와 `software-engineering`의 책임 경계가 반복해서 충돌할 때 독립 스킬 승격을 검토한다.
