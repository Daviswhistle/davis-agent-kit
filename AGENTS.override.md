# Davis Agent Kit 프로젝트 지침

## 적용 범위

1. 이 파일은 `davis-agent-kit` 저장소 안에서 작업할 때만 적용되며, 전역 `AGENTS.md`를 보완한다.
2. `davis-agent-kit`의 목적은 에이전트가 사용자의 판단 기준, 우선순위, 검증 습관을 따라 사고하고 행동하게 만드는 것이다.
3. 전역 철학, 핵심 원칙, 기본 동작, 행동 권한, 중단 조건, 스킬 라우팅의 규범 원본은 `AGENTS.md`다.
4. 이 저장소에만 필요한 관리 규칙, 설치 방식, 검증 계약, 파일 배치는 프로젝트 지침과 해당 실행물에 둔다.

## 저장소 구조

1. 가이드라인, 체크리스트, 템플릿, 테스트, 폴백 규칙, 설치 방식은 전역 원칙을 실행하는 현재 버전으로 관리한다.
2. 새 기준을 추가할 때는 철학, 핵심 원칙, 기본 동작, 적용 지침, 실행물 중 어디에 속하는지 먼저 분류한다.
3. 반복 가능한 교훈이나 체크리스트 항목이 생기면 `guidelines/`, `checklists/`, `templates/`, `skills/`, `user-model/` 중 맞는 위치에 반영할지 검토한다.
4. 교훈을 반영할 때는 불필요한 것을 제거하거나 필요한 것을 통합하는 데 도움이 되는지 먼저 본다.
5. 기준 원본 경로는 `${CODEX_HOME:-$HOME/.codex}/davis-agent-kit`이며, 이 경로는 저장소 전체를 가리켜야 한다.

## 수정 전 확인

1. 저장소를 수정하기 전에는 원격과 현재 작업 상태를 확인해 양쪽 변경을 보존할 수 있는 기준점을 만든다.
2. 워킹트리에 변경이 있으면 변경 주체와 목적을 확인하고, 불분명한 변경을 임의로 덮거나 숨기지 않는다.
3. 원격과 로컬이 갈라져 있으면 양쪽 의도를 확인해 충돌과 회귀를 막는다.
4. 파일의 역할이나 관리 방식을 말하기 전에는 해당 파일의 기존 설명과 사용 전제를 읽는다.

## 수정 계약

1. `AGENTS.md`에는 전역 지침 역할을 넘는 저장소 전용 절차를 넣지 않는다.
2. 이 저장소에만 적용되는 자동 로드 지침은 `AGENTS.override.md`에 둔다.
3. `skills/<skill-name>/` 아래의 스킬은 독립 설치 가능한 단위여야 한다. 필요한 지침, reference, helper, tests는 가능한 한 해당 스킬 폴더 안에 둔다.
4. 지침을 변경하면 직접 연결된 README, manifest, 설치·doctor 계약, 테스트, 스킬 라우팅의 정합성을 함께 확인한다.
5. 프롬프트나 스킬 변경은 작동 중인 전체 스택을 한꺼번에 다시 쓰지 않는다. 기준선을 남기고 한 번에 하나의 의미 있는 변경 묶음을 적용한 뒤 같은 대표 평가를 다시 실행한다.
6. `private/`, `scratch/`, `*.local.md`, `*.secret.md`, 토큰, 키, 비공개 원문 전문, 저작권 문제가 있는 긴 발췌는 공개 대상이 아니다.
