# 번역 QA 체크리스트

## 시작 전

- 이번 번역의 목표와 완료 조건을 한 문장으로 적었는가
- 원문 추출, 번역 단위, 화자 지도, 청크, 조립, 최종 산출물, QA helper 흐름을 확인했는가
- 이전 결과물이나 기억이 없어도 적용할 수 있는 기준을 확인했는가

## 형식

- 화자는 바뀔 때에만 표시했는가
- 화자와 첫 문단 사이에 빈 줄이 없는가
- 문장마다 줄을 띄우지 않고 문단 기준으로 나눴는가
- 링크와 이태릭체가 유지되었는가
- 독자용 문서에 원문 추출 라벨, 내부 메타데이터, 비영어 발언 placeholder가 남지 않았는가
- 설명 주석은 첫 등장 위치에만 있는가
- 설명 주석은 실제 오해를 막기 위한 것이며, 근거가 QA에 남아 있는가
- 최종 산출물이 복사-붙여넣기에 적합한가

## 의미와 흐름

- 화자 라벨은 추출 라벨이 아니라 실제 독자에게 필요한 발화자를 보여주는가
- 통역된 발언은 가능한 경우 통역자가 아니라 원 발언자에게 귀속했는가
- 사회자, 진행자, 임원, 애널리스트, 통역자의 역할이 섞이지 않았는가
- 한국어 높임말이 의도하지 않은 위계나 어색한 의전을 만들지 않았는가
- 반복되는 인사, 자기소개, 감사 표현은 의미를 잃지 않는 선에서 자연스럽게 정리했는가

## 숫자와 용어

- 숫자와 가이던스를 원문과 대조했는가
- 반복되는 숫자 가이던스는 대표 문장 하나가 아니라 모든 발생 위치를 확인했는가
- source unit과 최종 HTML `data-unit`을 대조할 수 있으면 같은 unit끼리 확인했는가
- 통화, `billion`, `억`, bp, 퍼센트, 회계연도와 달력연도 관계가 한국어 독자에게 같은 규모로 읽히는가
- 도메인 용어가 실제 사업 관계를 보존하는가

## 검수

- 어색한 직역 표현을 검색했는가
- 개념 검수 에이전트 또는 별도 검수 패스로 독자 관점의 실패 유형을 찾았는가
- helper 통과를 게시 가능 상태와 혼동하지 않았는가
- 실행한 검증, 생략한 검증과 사유, 남은 리스크를 분리해 기록했는가
