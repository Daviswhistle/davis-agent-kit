# 원칙

이 레포의 원칙은 [PHILOSOPHY.md](PHILOSOPHY.md)의 철학을 작업에서 반복 사용할 수 있는 판단 기준으로 바꾼 것입니다.

실행물은 원칙을 현재 환경에 적용하는 문서와 도구입니다. `AGENTS.md`, `GEMINI.md`, `guidelines/`, `checklists/`, `templates/`, `skills/`, 테스트, 폴백 규칙은 실제 작업과 검증 결과에 따라 계속 수정합니다.

## 원칙 승격 기준

어떤 기준을 원칙으로 올리기 전에 아래 조건을 확인합니다.

1. 여러 작업, 도구, 파일 형식에 적용된다.
2. 새 사례를 만날 때 별도 규칙 추가를 줄인다.
3. 변하지 않을 기준으로 둘 만큼 안정적이다.
4. 실행물의 방향을 정한다.
5. 작업자의 판단을 바꾼다.
6. 짧은 문장으로 반복 사용할 수 있다.

## 핵심 원칙

1. 불필요한 것은 제거한다.
2. 필요한 것은 통합한다.
3. 복잡하지만 필요한 것은 숨기거나 단계화한다.
4. 남기는 것은 이름, 위치, 형식이 역할을 드러내게 한다.
5. 수치나 지표를 해석하기 전에 측정 대상이 실제로 존재했고 관측 가능했는지(분모의 현실 전제)를 먼저 의심한다.
6. 제거하거나 통합한 뒤에도 정확성, 안전성, 맥락, 신뢰가 유지되는지 검증한다.

## 실행물의 역할

1. `guidelines/`는 원칙을 특정 작업 표면에 적용하는 해설이다.
2. `checklists/`는 완료 전에 놓치기 쉬운 조건을 확인하는 도구다.
3. `skills/`는 반복 작업을 재현 가능한 절차와 리소스로 묶은 실행 단위다.
4. `templates/`와 helper는 원칙을 산출물 형식이나 자동화로 고정한 것이다.
5. 실행물이 계속 늘어나면 원칙이 충분히 단순하거나 강하지 않은지 다시 본다.
